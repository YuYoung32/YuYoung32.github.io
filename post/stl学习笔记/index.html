类型萃取 iterator_traits

一个算法需要得知容器的属性时，具体来说，是需要知道某些指标，比如：

iterator_category 是什么类型的,比如说双向遍历类型
value_type		容器内装的值是什么类型的,比如说int
difference_type	容器内距离的类型,比如说long int
pointer			指针类型是什么,比如说int*
reference		引用类型是什么,比如说int&

怎么实现呢？

具体而言，如何知道根据模板传入的类型参数做出判断。

一个错误的想法

c++11提供了decltype()做类型推断，可以用decltype(_Ty)==decltype(int*)做判断然后具体赋值吗？显然不可能，c++不提供类型间的操作。不能比较int==int，必须要实例化比如int()==int()，而==的前提是左右对象类型相同，不相同的情况下会编译报错。

这里要和golang的类型推断语法比较一下，golang的类型推断 switch val.type，然后就可以判断类型是否符合了。

还有一种方法，c++为了能够根据传入类型做操作，天然支持了重载，自动根据类型调用不同的函数。难道说要在容器里定义函数，然后让外部调用函数获取属性吗？好像也可以，但是这样还需要调用函数，会降低性能。并且，不符合泛型编程数据和方法分开的准则。接下来看看STL是怎样实现的。

STL是这样实现的

  1. 每个容器自己有一个_xx_iterator模板类，使用typedef，把自己的属性写死，比如下面是stl_list的源码，把自己的iterator_category写死为bidirectional_iterator_tag。
template<typename _Tp>
struct _List_iterator
{
	...
    typedef ptrdiff_t                          difference_type;  //ptrdiff_t 到最后就是long int
    typedef std::bidirectional_iterator_tag    iterator_category; 
    //std::bidirectional_iterator_tag到最后就是  struct input_iterator_tag { };
    typedef _Tp                                value_type;
    typedef _Tp*                               pointer;
    typedef _Tp&                               reference;
    ...
};

这样外部就可以直接像这样获取到类型属性，获得这个属性的对象。

//获取到属性对象
list<int>::iterator::iterator_category listCat;
vector<int>::iterator::iterator_category vectorCat;

至此，完成了属性类型typename到一个属性对象object的转换。

  1. 还有一个问题，不是所有iterator都是模板，比如指针，引用。这时候也想获得他们的属性，怎么办呢。很自然的想到使用适配器模式。若传入的是指针或者引用，则进行一个partial specialization。若是普通类型，则读取他内置的属性，否则适配器直接返回属性。

    这个适配器是std::traits<iterator _It>，看看他是怎样进行偏特化的。

    传入普通类型

  template<typename _Iterator>
    struct __iterator_traits<_Iterator,
			     __void_t<typename _Iterator::iterator_category,
				      typename _Iterator::value_type,
				      typename _Iterator::difference_type,
				      typename _Iterator::pointer,
				      typename _Iterator::reference>>
    {
      typedef typename _Iterator::iterator_category iterator_category;
      typedef typename _Iterator::value_type        value_type;
      typedef typename _Iterator::difference_type   difference_type;
      typedef typename _Iterator::pointer           pointer;
      typedef typename _Iterator::reference         reference;
    };

  template<typename _Iterator>
    struct iterator_traits //从这里实例化，父类接到并读取传入iterator内置的
    : public __iterator_traits<_Iterator> { };

传入指针/引用类型

template<typename _Tp>
struct iterator_traits<_Tp*>
{
    typedef random_access_iterator_tag iterator_category;
    typedef _Tp                         value_type;
    typedef ptrdiff_t                   difference_type;
    typedef _Tp*                        pointer;
    typedef _Tp&                        reference;
};

/// Partial specialization for const pointer types.
template<typename _Tp>
struct iterator_traits<const _Tp*>
{
    typedef random_access_iterator_tag iterator_category;
    typedef _Tp                         value_type;
    typedef ptrdiff_t                   difference_type;
    typedef const _Tp*                  pointer;
    typedef const _Tp&                  reference;
};

外部怎样通过适配器获得属性的对象呢?

//使用适配器
iterator_traits<list<int>::iterator>::iterator_category listItAdapter;
iterator_traits<int*>::iterator_category intItAdapter;
//不使用适配器
list<int>::iterator::iterator_category listIt; //和前者都是std::bidirectional_iterator_tag对象
